# サンプルコード用のドメインの準備方法

ブロックチェーン上にサンプルコード用のドメインを準備する手順を説明します。  

## 概要

サンプルコード実行環境をセットアップするときに、ステップ１を実施した後、
ブロックチェーンの管理ユーザに連絡してサンプルコード用にドメインを準備してもらいます。
準備されたドメインはステップ１で作成したウォレットが管理権限を持つように構成されます。
したがって、サンプルコードからそのドメインを自由に利用することができるようになっています。

ブロックチェーンの管理ユーザにサンプルコード用のドメインの作成を依頼するとき、
ステップ１で作成したウォレットのアドレスを管理ユーザに伝える必要があります。
ウォレットのアドレスは設定ファイル（`etc/config.json`）の`address`欄に記載されています。
管理ユーザは、作成したドメインのIDを依頼者に伝えます。このドメインIDは、セットアップのステップ３で使用します。  

## ドメインの再利用

ドメインを新規作成するのではなく、作成済みのドメインを再利用する場合には、管理ユーザに依頼せずに自分でドメインを準備することができる場合があります。その条件は、再利用するドメインの管理権限を有していることです。自分でドメインを準備するには、下記の手順のうち、ドメインの新規作成の手順をスキップして実施します。  

## ドメインの準備の手順

この手順は、基本的にはブロックチェーンの管理ユーザが実施します。  
ただし、ドメインを作成した以後の手順は、ドメインの管理権限を有したユーザであれば実施可能です。  

### 前提条件

サンプルコード実行環境のセットアップのステップ１が完了しており、ステップ１で作成したウォレットのアドレスを把握していること。

### 1. ドメインの新規作成

c1domainコントラクトを使ってドメインを新規に作成します。

### 2. ユーザの作成

上で作成したドメイン内にc1createでユーザを作成します。

### 3. ユーザへのアドレスの登録

上で作成したユーザにステップ１で作成したウォレットのアドレスを紐づけます。

### 4. ユーザをドメインの管理ユーザに追加する

上で作成したユーザを、上で作成したドメインの管理ユーザとして登録します。

## スマートコントラクト

上記手順をスマートコントラクトで実行する場合のコードを下記に示します。  

```
function create_samples_domain(address){
    var domainID = openContract('c1domain').call();
    var userID = openContract('c1create').call({type: 'user', domain: domainID});
    openContract('c1update').call({id: userID, prop: 'wallets', value: [address]});
    openContract('c1update').call({id: domainID, prop: 'add managed_by', value: [userID]});
    return domainID;
}
```
